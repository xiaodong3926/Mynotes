在Linux中，**一切皆文件！**
## 一、基础文件
正如上面的那句话，在Linux中无论是硬件还是软件，本质上都是文件。
目录文件：目录实际存放的是指向该文件的链接与文件名，当删除文件时，指向该文件的链接数会减1,直到减到0,就意味着该文件不会再被使用，存放此文件的硬盘空间会被标记为可用
## 二、系统调用与库函数
#### 系统调用

**系统内核**：就是一组对底层硬件的驱动程序

**系统调用**：由系统内核提供的，用来对文件与设备进行访问的函数
#### 库函数

为什么会有库函数呢？原因是系统调用函数会影响系统性能，而减少性能开销的方法为减少系统调用次数，一次性让系统调用函数尽可能的多读取数据，因此Linux提供了一系列标准的函数库，就比如说提供输出缓冲功能的I/O库(对应C语言的头文件stdio.h)，你可以输出任意长度的数据

**缓冲区**：缓冲区存储的是低速设备的数据，然后高速设备可以直接读取这些数据不必去等待低速设备，这减少了系统调用，而且尽可能的让一次调用读取了更多的数据、

## 三、文件访问的底层原理

一个在运行的程序被称为**进程**，而每个进程都有对应的**文件描述符** ，一般都是从整数3开始

三个基础描述符:

| 0    | 1    | 2    |
| ---- | ---- | ---- |
| 标准输入 | 标准输出 | 标准错误 |
`ls -l /proc/$$/fd/`用此命令来查看当前shell进程的文件描述符
`ls -l /proc/<进程ID>/fd/`用此命令来查看指定进程的文件描述符

#### write系统调用:
作用是将缓冲区（buffer）的前n个字节的数据写入与文件描述符相关联的文件中，返回的是实际写入的字节数（0代表未写入任何数据，-1代表出现错误）
``` c
#include<unistd.h>
size_t write(int fildes,const void *buf,size_t nbytes);
```
(size_t表示无符号整数，专门用来表示数据的大小)

#### read系统调用：
把与文件描述符相关联的文件中读取的n个字节的数据写入缓冲区，返回实际读入的字节数
```C
#include<unistd.h>
size_t read(int files,void *buf,size_t nbytes);
```

在写入操作后，文件指针处于文件末尾的位置，此时想要读取，要用`lseek(fd, 0, SEEK_SET)`来从头读取文件

#### open系统调用：
用于创建一个新的文件描述符，当两个程序共同打开同一文件则会返回两个不同的文件描述符，两个程序的改动会覆盖在文件上
```C
#include<fcntl.h>
#include<sys/types.h>
#include<sys/stat.h>
int open(const char *path,int oflags);
int open(const char *path,int oflags,mode_t mode);
```
(mode_t同样是无符号整数类型，它用于表示文件权限与类型)

oflags的三个基础的写入模式(只能选择一个)：

| O_RDONLY | O_WRONLY | O_EDWARD |
| :------: | :------- | :------- |
|    只读    | 只写       | 读写       |
oflags参数还可以选择下列组合(用"|")：

| O_APPEND | O_TRUNC | O_CREAT       | O_EXEL       |
| -------- | ------- | ------------- | ------------ |
| 在文件末尾写入  | 丢弃已有内容  | 按照mode参数来创建文件 | 与O_CREAT一起使用 |
mode参数的可选项（数字为0时表示无任何权限）：

| 宏定义       | 权限类型 | 适用对象  | 代表数字 |
| --------- | ---- | ----- | ---- |
| `S_IRUSR` | 读    | 文件属主  | 4    |
| `S_IWUSR` | 写    | 文件属主  | 2    |
| `S_IXUSR` | 执行   | 文件属主  | 1    |
| `S_IRGRP` | 读    | 文件所属组 | 4    |
| `S_IWGRP` | 写    | 文件所属组 | 2    |
| `S_IXGRP` | 执行   | 文件所属组 | 1    |
| `S_IROTH` | 读    | 其他用户  | 4    |
| `S_IWOTH` | 写    | 其他用户  | 2    |
| `S_IXOTH` | 执行   | 其他用户  | 1    |
属主：文件所有者
属组：文件所属组的成员

mode参数在open调用中实际上是发出权限设置的请求，所请求的权限是否被设置则取决于**umask值**
#### umask值
umask值由3位8进制数组成，当新文件被创建时，umask作为掩码为文件设置权限
**掩码：** 通过特定的规则来起到筛选、过滤作用的模板与过滤器
![[Pasted image 20260511081350.png]]
*每个数字的含义（第一个数字表示主用户，第二个数字代表组用户，第三个数字表示其他用户）*

如果要禁止组用户的写权限与执行权限，同时禁止其他用户的写权限，那么第一个数字是”0“,第二个数字是”2|1“（也就是3），第三个数字是”2“，因此，此umask值是032

umask对mode请求权限的影响是在umask取反之后与mode参数请求权限的二进制进行与运算,最终得到的数就是该文件的真实权限

如果mode请求的权限是666,而umask是022,那进行计算之后，最终该文件的权限是644
#### close系统调用
close的作用是终止文件描述符与文件相关联，释放后的文件描述符能够被重新使用，成功返回0,出错返回-1
```C
#include<unistd.h>
int close(int files);
```
#### ioctl系统调用
ioctl是用于控制硬件设备的系统调用，每个设备都有自己的ioctl
