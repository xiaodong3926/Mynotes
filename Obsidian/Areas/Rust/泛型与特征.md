# 泛型与特征

### 1. 泛型 Generics

` gen_spec_i32(SGen(32) ` 隐式地指定类型参数`i32`

` generic::<char>(SGen('a')); ` （SGen是泛型）显式地指定类型参数 `char`

它们的区别仅仅是显式要写 ::<具体类型>

泛型的应用的语法规范

```struct
struct Point<T, U> {
    x: T,
    y: U,
}
impl<T, U> Point<T, U> {
    fn mixup<M,N>(self,other:Point<M,N>)->Point<T,N>{
        Point{
            x:self.x,
            y:other.y,
        }
    }
}
fn main() {
    let p1 = Point { x: 5, y: 10 };
    let p2 = Point { x: "Hello", y: '中'};
    let p3 = p1.mixup(p2);
    assert_eq!(p3.x, 5);
    assert_eq!(p3.y, '中');
}
```

上段的代码中，` mixup<M,N>` 中的M，N是为 `other` 绑定的**新的**类型参数 **`impl<T, U>`** 是为 `self` 绑定类型参数，之后返回值 `Point<T,N>` 中T(u32)与N(char)一个是来自self,一个是来自other。
`impl<T, U> Point<T, U>`  的意思是对于所有的类型 T，为 Point<T, U> 实现以下方法
### 2. Const泛型

在之前的泛型中，可以抽象为一句话：针对类型实现的泛型，所有的泛型都是为了抽象不同的类型，那有没有针对值的泛型？答案就是 `Const 泛型`。

```
fn display_array<T: std::fmt::Debug, const N: usize>(arr: [T; N]) {
    println!("{:?}", arr);
}
fn main() {
    let arr: [i32; 3] = [1, 2, 3];
    display_array(arr);
    let arr: [i32; 2] = [1, 2];
    display_array(arr);
}
```

`std::fmt::Debug` 用于打印调试信息, `println!("{:?}", arr); ` ***：？*** 这个格式化符号要求参数实现 Debug trait
数组大小必须是编译时常量，因此在编译时就要知道其大小，因此不同长度的数组并不是统一类型，在上段代码中，T仍然是泛型，而N是泛值

### 3. 特征 Trait

特征定义了 **一组可以被共享的行为，只要实现了特征，你就能使用这组行为**

```
pub trait Summary {
    fn summarize(&self) -> String;
}
```

这个代码的意思是，定义了一个名为Summary的trait,任何实现了该特征的类型都能使用summarize这个方法。

这是特征的另一个用法: **使用特征作为函数参数** 

```
pub fn notify(item: &impl Summary) {
    println!("Breaking news! {}", item.summarize());
}
```

你可以使用任何实现了 `Summary` 特征的类型作为该函数的参数，同时在这个函数内，你还可以调用它的方法 ` summarize`，实际上面的代码是一个语法糖，完整形式如下：

```
pub fn notify<T: Summary>(item: &T) {
    println!("Breaking news! {}", item.summarize());
}
```

而 `<T:Summary>`正是所谓的 **特征约束** ,它的意思是T 类型必须实现 ` Summary trait`

**使用特征作为函数返回值** 

` fn random_animal(random_number: f64) -> impl Animal{}` 

如上，就是示例,具体用途有待商榷!

```
std::ops::Sub
impl<T: Sub<Output=T>> Sub for Point<T>{}
```
Sub是一个内置特征,在使用时要加上std::ops::Sub

`impl<T: Sub<Output=T>>` 表示**类型 `T` 必须实现 `Sub` trait，并且 `T - T` 的结果必须是 `T` 类型**

### 4. 特征对象

```
fn returns_summarizable(switch: bool) -> impl Summary {
    if switch {
        Post {
           // ...
        }
    } else {
        Weibo {
            // ...
        }
    }
}
```

上面这个代码是错的，原因是 `impl Trait` 的返回值类型并不支持多种不同的类型返回,而为了解决这种问题便创造除了 **特征对象** 

如果要编译成功用特征对象来写，如下：

```
fn returns_summarizable(switch: bool) -> Box<dyn Summary> {
    if switch {
        Box::new(Post {
            // ...
        })
    } else {
        Box::new(Weibo {
            // ...
        })
    }
}
```

`dyn`: 关键字,表示动态分发
`Summary` :表示任何实现了 Summary 特征的类型(`dyn Summary` 是一个类型)
`dyn Summary` 本身是没有固定大小的（因为编译器不知道具体是 Post 还是 Weibo）
这就是为什么我们必须把它放在指针后面，比如` &dyn Summary ` 或 `Box<dyn Summary>` 。我们不能直接拥有 `dyn Summary`  类型的变量，只能拥有指向它的指针（胖指针）
为什么先前不能编译，而加一个**Box** 就行了呢？ 因为此时它们都被视为`Box<dyn Summary>` 类型

**对象安全**

```
trait MyTrait {
    fn f(&self) -> Box<dyn MyTrait>;
}
impl MyTrait for u32 {
    fn f(&self) -> Box<dyn MyTrait> { Box::new(42) }
}
impl MyTrait for String {
    fn f(&self) -> Box<dyn MyTrait> { Box::new(self.clone()) }
}
fn my_function(x: Box<dyn MyTrait>)->Box<dyn MyTrait> {
    x.f()
}
fn main() {
    my_function(Box::new(13_u32));
    my_function(Box::new(String::from("abc")));
    println!("Success!")
}
```

在这里`fn f(&self) -> Box<dyn Mytrait>` 不能写成 `fn f(&self) -> Self` 

1.方法的返回类型不能是 Self
2.方法没有任何泛型参数

对下面一个语句的解释应该是:泛型(静态分发)是编译时就确定其类型，而特征对象(动态分发)则要到运行时才知道其类型，两者会冲突

### 5.深入了解

**关联类型** 

```
pub trait Iterator {
    type Item;//type是关键字
    fn next(&mut self) -> Option<Self::Item>;
}
```

如上所示。这里定义了一个名为Item的关联类型，然后创建了一个next方法，**返回类型是可选的当前调用者的Item类型**

`type Item = u32` ——这就是 **指定关联类型的语法** 

```
impl <T: Sub<Output=T>> Sub for Point<T> {
    type Output = Self;
```
这里为什么有两个Output呢？首先，第一个Output声明了T必须实现了Sub特征的关联类型是T,第二个声明了当Point使用Sub特征时，其关联类型是Self

**错误写法** 

```
trait Contains {
    type A;
    type B;
    fn contains(&self, _: &A, _: &B) -> bool;          
}
```

原因:A 和 B 是关联类型，不是独立的类型，必须通过 `Self::A` 和 `Self::B` 来引用它们

A、B本身不是类型名，它们是关联类型的名字，Self::A 、Self::B才是具体的类型

**默认泛型参数**

在使用泛型参数时，可以为此泛型指定一个具体的类型

```
use std::ops::Add;

struct Millimeters(u32);
struct Meters(u32);

impl Add<Meters> for Millimeters {
    type Output = Millimeters;

    fn add(self, other: Meters) -> Millimeters {
        Millimeters(self.0 + (other.0 * 1000))
    }
}

```

`Add<Meters>` 指定了泛型参数的类型为`Meters`
但如果没有指定默认的泛型参数类型是Meters,就会编译错误，因为默认则为相同类型相加
```
use std::ops::Sub;
#[derive(Debug, PartialEq)]
struct Point<T> {
    x: T,
    y: T,
}
impl<T:Sub<Output=T>> Sub<Point<T>> for Point<T> {
    type Output = Self;
    fn sub(self, other: Self) -> Self::Output {
        Point {
            x: self.x - other.x,
            y: self.y - other.y,
        }
    }
}
fn main() {
    assert_eq!(Point { x: 2, y: 3 } - Point { x: 1, y: 0 },
        Point { x: 1, y: 3 });
    println!("Success!")
}
```
这是一个经典的例子，`Sub<Point<T>>` 为操作符右边的参数限定了为`Point<T>` 类型，
`for Point<T>` 则限定了操作符左边的类型，`impl<T:Sub<Output=T>>` 说明T泛型必须满足Sub特征且输出类型为T。由于两边是相同类型所以还可以写成`Sub<Self>`、`Sub`.

**调用同名的方法**

如果当特征方法与结构体方法重名，当调用这个重名的方法时，编译器会优先调用结构体方法，那么该如何调用特征方法呢？需要用到显式调用的语法：
```
fn main() {
    let person = Human;
    Pilot::fly(&person); // 调用Pilot特征上的方法
    Wizard::fly(&person); // 调用Wizard特征上的方法
    person.fly(); // 调用Human类型自身的方法
}
```
因为 fly 方法的参数是 **self**，当显式调用时，编译器就可以根据调用的类型( **self 的类型**)决定具体调用哪个方法

如果重名的是关联函数：
这样写：`Pilot::fly()`就会报错，原因是编译器不知道是哪一个类型实现了Pilot特征，想要成功编译，就需要使用**完全限定语法**：
`<Dog as Animal>::baby_name()` **as**关键字向编译器提供了类型注解

**特征定义中的特征约束**
```
use std::fmt::Display;
trait OutlinePrint: Display {
    fn outline_print(&self) {
        let output = self.to_string();
        let len = output.len();
        println!("{}", "*".repeat(len + 4));
        println!("*{}*", " ".repeat(len + 2));
        println!("* {} *", output);
        println!("*{}*", " ".repeat(len + 2));
        println!("{}", "*".repeat(len + 4));
    }
}
```
`OutlinePrint: Display` 这就是所谓的在**特征定义中的特征约束** ，其含义是**任何想要实现 OutlinePrint这个 trait 的类型，也 必须 同时实现 std::fmt::Display 这个 trait。** 注意：特征约束是在特征与特征之间！

```
trait Person {
    fn name(&self) -> String;
}

trait Student: Person {
    fn university(&self) -> String;
}

trait Programmer {
    fn fav_language(&self) -> String;
}

supertraits.
trait CompSciStudent: Programmer + Student {
    fn git_username(&self) -> String;
}

fn comp_sci_student_greeting(student: &dyn CompSciStudent) -> String {
    format!(
        "My name is {} and I attend {}. My favorite language is {}. My Git username is {}",
        student.name(),
        student.university(),
        student.fav_language(),
        student.git_username()
    )
}

struct CSStudent {
    name: String,
    university: String,
    fav_language: String,
    git_username: String
}

impl Person for CSStudent{
    fn name(&self) -> String{
        self.name.clone()
    }
}
impl Student for CSStudent{
    fn university(&self) -> String{
        self.university.clone()
    }
}
impl Programmer for CSStudent{
    fn fav_language(&self) -> String{
        self.fav_language.clone()
    }
}
impl CompSciStudent for CSStudent{
    fn git_username(&self) -> String{
        self.git_username.clone()
    }
}

fn main() {
    let student = CSStudent {
        name: "Sunfei".to_string(),
        university: "XXX".to_string(),
        fav_language: "Rust".to_string(),
        git_username: "sunface".to_string()
    };

    println!("{}", comp_sci_student_greeting(&student));
}
```
上面是来自练习的一道题目，为梳理清楚各个概念，请看下面这段代码：
```
struct CSStudent {
    name: String,
    university: String,
    fav_language: String,
    git_username: String
}

impl Person for CSStudent{
    fn name(&self) -> String{
        self.name.clone()
    }
}
impl Student for CSStudent{
    fn university(&self) -> String{
        self.university.clone()
    }
}
impl Programmer for CSStudent{
    fn fav_language(&self) -> String{
        self.fav_language.clone()
    }
}
impl CompSciStudent for CSStudent{
    fn git_username(&self) -> String{
        self.git_username.clone()
    }
}
```
就拿` impl Person for CSStudent` 这段来说，明明下文中都没用到`self.name`为什么还要写呢？因为在Person特征中有这样一行代码:`fn name(&self) -> String;` 要返回一个拥有所有权的String类型，所以要写，而且因为`self.name` 是访问self的字段，所以还要加`.clone()` 

`CompSciStudent: Programmer + Student` 与`Student: Person` 都有特征约束，因此要把对应的约束都实现了

`fn comp_sci_student_greeting(student: &dyn CompSciStudent)` 对于这一行：为什么`println!("{}", comp_sci_student_greeting(&student))` 的参数只用填`&student`？(dyn见4节——特征对象) 这是因为student是CSStudent类型，已经实现了CompSciStudent特征，因此编译器会自动转化为`&dyn CompSciStudent`

**在外部类型上实现外部特征(newtype)** 
在此之前，先介绍一个概念:**孤儿规则**，简单来说，就是特征或者类型必需**至少有一个是本地的**，才能在此类型上定义特征。
`impl<T> std::fmt::Display for Vec<T> {}` 在这段代码中，Display与Vec都来自标准库，无一个本地的，因此编译器会报错。因此需要一个办法来绕过孤儿规则，那就是使用newtype 模式，简而言之：就是为一个元组结构体创建新类型。该元组结构体封装有一个字段，该字段就是希望实现特征的具体类型。
```
struct Wrapper(Vec<String>);
impl fmt::Display for Wrapper {
 fn fmt(&self, f: &mut fmt::Formatter) -> fmt::Result {
        write!(f, "[{}]", self.0.join(", "))
    }};
```
`struct Wrapper(Vec<String>) ` 就是一个元组结构体，它定义了一个新类型 Wrapper

`self.0.join(", ")` 的意思就是从Wrapper中取出数组Vec然后再执行join方法,Newtype 模式在编译后通常会被优化掉,Wrapper(Vec<String>) 在运行时和直接使用 Vec<String> 没有性能区别，它纯粹是编译期的类型安全外壳。这体现了 Rust 的“零成本抽象”特性。
